# Рефакторинг структури проєкту

## Створені компоненти

### Layouts
- **BaseLayout.astro** - Базовий layout з HTML структурою, мета-тегами та глобальними стилями

### Components
- **Header.astro** - Хедер з логотипом та кнопками (оновити, вийти)
- **ErrorMessage.astro** - Універсальний компонент для відображення помилок (error/warning)
- **NotificationCard.astro** - Картка окремого сповіщення
- **NotificationsList.astro** - Контейнер для списку сповіщень з заголовком та кнопкою "Всі прочитані"

## Оновлені сторінки

### index.astro (Сторінка входу)
- Використовує BaseLayout
- Використовує ErrorMessage компонент
- Зменшено дублювання коду

### notifications.astro (Сторінка сповіщень)
- Використовує BaseLayout
- Використовує Header компонент
- Використовує ErrorMessage компонент
- Використовує NotificationCard для кожного сповіщення
- Використовує NotificationsList як контейнер
- JavaScript логіка залишилась на сторінці (для доступу до DOM)

## Переваги рефакторингу

1. **Повторне використання коду** - компоненти можна використовувати на різних сторінках
2. **Легше підтримувати** - зміни в одному компоненті застосовуються скрізь
3. **Чистіший код** - сторінки стали коротшими та зрозумілішими
4. **Консистентність** - однаковий дизайн та поведінка на всіх сторінках
5. **Типізація** - Props інтерфейси забезпечують type safety
